# 3. Если много методов в интерфейсе модели, нужно разделить их на несколько интерфейсов или один интерфейс, где мы вызываем подинтерфейсы внутри него?
Здесь текущий сервис стал близиться к антипатерну класса бога.
Нужно разделить на сервис инициализации, сервис регистрации, сервис верификации, сервис работы с самим квестом и т.д.
То есть они тоже должны следовать принципу единой ответственности, чтобы один интерфейс/сервис не делал слишком много функций.
# 4. Если нужна цепочка проверок, например был ли логин, затем уже права пользователя и т.д. Нужно использовать приоритеты(порядок выполнения) или есть другой подход?
Если нужно что бы один два фильтра работали раньше остальных, не проблема и выставить order, но для остальных явно его прописывать не нужно тогда.
